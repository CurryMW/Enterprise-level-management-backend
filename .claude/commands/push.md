你是一个 Git 提交助手。请按以下步骤执行：

## 步骤 1：分析变更

并行执行以下命令：

- 运行 `git status` 查看所有未跟踪和已修改的文件
- 运行 `git diff` 和 `git diff --cached` 查看具体变更内容
- 运行 `git log --oneline -5` 查看最近的提交记录，以便保持提交风格一致

## 步骤 2：生成提交信息

根据变更内容，生成一条简洁的中文 commit 信息，遵循以下规范：

- 格式：`<type>: <描述>`
- type 类型：
  - `feat` — 新功能
  - `fix` — 修复 bug
  - `docs` — 文档变更
  - `style` — 代码格式（不影响逻辑）
  - `refactor` — 重构
  - `perf` — 性能优化
  - `test` — 测试相关
  - `chore` — 构建/工具变更
  - `init` — 初始化
- 描述用中文，简洁概括本次变更的目的
- 如果涉及多个变更，用最主要的变更作为提交信息

## 步骤 3：确认并提交

1. 向用户展示将要提交的文件列表和生成的 commit 信息，询问用户确认
2. 用户确认后，将相关文件添加到暂存区（注意排除 .env、credentials 等敏感文件）
3. 执行 `git commit`，commit 信息末尾附加：`Co-Authored-By: Claude Opus 4.6 <noreply@anthropic.com>`

## 步骤 4：推送到远程仓库

1. 检查当前分支是否有对应的远程跟踪分支
2. 如果没有，使用 `git push -u origin <branch>` 推送并设置上游
3. 如果有，使用 `git push` 推送
4. 推送完成后，展示推送结果

## 注意事项

- 如果没有任何变更，告知用户无需提交
- 如果有冲突或推送失败，告知用户具体原因并给出建议
